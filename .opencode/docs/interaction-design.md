## 1. 信息架构（IA）

### 1.1 Tab 结构

采用 **3 Tab** 结构（参考 Bark 的首页/消息列表/设置布局），简洁清晰：

```
┌───────────────────────────────────────────────────────┐
│                    TabBarController                    │
│                                                        │
│   Tab 0          Tab 1          Tab 2                  │
│  ┌─────┐       ┌─────┐       ┌─────┐                 │
│  │ 首页 │       │ 收信 │       │ 设置 │                 │
│  │ 🏠  │       │ 📬  │       │ ⚙️  │                 │
│  └─────┘       └─────┘       └─────┘                 │
│   MainVC      InboxVC      SettingsVC                 │
└───────────────────────────────────────────────────────┘
```

| Tab | 名称 | 图标 (SF Symbol) | 核心职责 | 优先级 |
|-----|------|-------------------|----------|--------|
| 0 | 首页 | `house` / `house.fill` | 应用卡片入口、快速操作、Token 复制 | P0 |
| 1 | 收信箱 | `tray` / `tray.fill` | 消息列表、分组、搜索、消息路由 | P0 |
| 2 | 设置 | `gearshape` / `gearshape.fill` | 配置、Token 管理、调试面板入口 | P0 |

### 1.2 页面层级

```
TabBarController
├── Tab 0: 首页 (MainViewController)
│   ├── QR 扫码页 (QRScannerViewController)
│   ├── Web 浏览器页 (WebBrowserViewController) ← 核心页面
│   ├── 口令检测 (后台自动触发，无独立页面)
│   └── 应用详情页 (AppDetailViewController)
│
├── Tab 1: 收信箱 (MessageInboxViewController)
│   ├── 消息详情页 (MessageDetailViewController)
│   ├── 搜索页 (MessageSearchViewController)
│   └── Web 浏览器页 (从消息路由跳转)
│
└── Tab 2: 设置 (SettingsViewController)
    ├── 服务器配置 (ServerConfigViewController)
    ├── Token 管理 (TokenManageViewController)
    ├── Token 生成 (TokenGenerateViewController)
    ├── API Key 管理 (APIKeyManageViewController)
    ├── API Key 示例 (APIKeyExampleViewController)
    ├── 缓存管理 (ManagementViewController → CacheManagementViewController)
    ├── 收藏管理 (ManagementViewController → FavoriteViewController)
    ├── WebAccess (WebAccessViewController)
    ├── 关于 (AboutViewController)
    └── Debug 面板 (DebugPanelViewController) ← 仅 DEBUG
        ├── Handler 调试 (HandlerTestViewController)
        └── 缓存测试 (ManifestCacheTestViewController)
```

### 1.3 核心用户旅程

#### 旅程 1: 快速复制 Token 并发送通知

```
打开 App → 首页 → 看到应用卡片 → 长按卡片 → 复制推送 Token → 粘贴到外部工具发送通知
```

交互要素：卡片长按 → Action Sheet → "复制 Token" 一键操作
反馈：HUD 显示 "已复制"

#### 旅程 2: 接收推送 → 路由到目标页面

```
收到推送 → Tab 1 收信箱 badge +1 → 打开收信箱 → 看到未读消息 → 点击消息 → 自动跳转到目标应用/URL
```

交互要素：Tab badge、消息未读红点、点击自动路由
反馈：消息标记已读、平滑跳转动画

#### 旅程 3: 扫码加载离线应用

```
首页 → 点击扫码按钮 → 扫描 Manifest QR → 自动解析下载 → 应用出现在首页卡片 → 点击打开
```

交互要素：扫码按钮固定在导航栏、下载进度条、卡片刷新动画

#### 旅程 4: 口令触发

```
复制口令到剪贴板 → 切回 App → 自动检测 → 弹出确认对话框 → 执行跳转
```

交互要素：后台静默检测、非侵入式确认弹窗、口令入口不在界面显眼位置

#### 旅程 5: 调试通知

```
设置 → Debug 面板 → 消息中心 Tab → 发送测试通知 → 收信箱实时显示
```

交互要素：Debug 面板隐藏入口、实时刷新消息列表

### 3.2 卡片设计

每个应用卡片（`URLGridCell`）展示信息：

| 元素 | 位置 | 说明 |
|------|------|------|
| 应用图标 | 卡片左上 | 使用 favicon 或首字母头像 |
| 应用名称 | 图标右侧 | 标题，最多 2 行，超出截断 |
| 缓存状态指示器 | 名称下方 | 彩色圆点 + 文字标签 |
| 最近访问时间 | 右上角 | 相对时间（"2分钟前"、"昨天"） |
| Token 快速复制 | 卡片底部 | 显示 Token 前 4 位 + "..." + 复制图标 |
| 访问次数 | 右下角（可选） | 小字显示 "3 次" |

#### 缓存状态标识（缓存可视化）

| 状态 | 图标 | 颜色 | 文字 |
|------|------|------|------|
| 持久化 (persistent) | `●` | `systemGreen` | "持久化" |
| 临时 (lazy) | `○` | `systemOrange` | "临时" |
| 离线可用 | `●` | `systemBlue` | "离线" |
| 需更新 | `↻` | `systemYellow` | "需更新" |
| 未缓存 | `○` | `systemGray` | "未缓存" |

### 3.3 首页交互

| 操作 | 触发方式 | 行为 |
|------|---------|------|
| 打开应用 | 单击卡片 | `WebBrowserManager.openBrowser()` 加载 URL |
| 快速复制 Token | 长按卡片 → Action Sheet | 复制该应用的推送 Token，HUD 提示 |
| 扫码添加 | 导航栏左侧扫码按钮 | 打开 `QRScannerViewController` |
| 查看详情 | 长按卡片 → "详情" | 打开 `AppDetailViewController` |
| 清理全部 | 导航栏右侧清理按钮 | 二次确认后清理所有历史和缓存 |
| 下拉刷新 | 下拉手势 | 刷新应用列表和缓存状态 |
| 口令检测 | 后台自动触发 | App 从后台切回前台时检测剪贴板 |

### 3.4 快速复制 Token（重点功能）

**目标**: 用户能在 3 秒内复制到应用的推送 Token

```
方式 1（推荐）: 长按卡片 → Action Sheet → "复制 Token"
方式 2: 卡片底部 Token 区域点击 → 直接复制
```

Action Sheet 内容：

```
┌────────────────────────────────────┐
│  应用名称                           │
│  域名: example.com                 │
│  缓存: 2.3 MB                      │
│  ────────────────────────────────  │
│  📋 复制推送 Token     ← 高亮推荐   │
│  📋 复制应用 URL                   │
│  📌 置顶 / 取消置顶                │
│  ⭐ 收藏                           │
│  🗑 清除缓存                       │
│  ℹ️ 查看详情                       │
│  ────────────────────────────────  │
│  取消                              │
└────────────────────────────────────┘
```

### 3.5 口令检测入口（低调设计）

口令（Command Parser）有入口但不需要太多展示：

- **主入口**: 后台自动检测（App 切回前台时检查剪贴板）
- **手动入口**: 设置页底部，一行文字链接 "检测剪贴板口令"
- **无独立页面**: 检测成功后直接弹出确认对话框
- **视觉权重**: 极低，不占首页显著位置

```
触发流程：
剪贴板检测到口令 → 顶部弹出确认条（非模态）→ 用户确认 → 执行跳转
```

### 3.6 QR 扫码入口

```
位置: 导航栏左侧按钮 (UIBarButtonItem)
图标: qrcode.viewfinder (SF Symbol)
行为:
  - 标准 URL → WebBrowserManager 打开
  - Manifest JSON URL → 自动解析并缓存
  - wb-app:// 自定义协议 → 路由处理
  - wbsk://command → 口令解析
  - 其他内容 → 提示无法识别
```

### 4.8 消息路由逻辑

消息点击后的路由行为：

```
点击消息
    │
    ├── payload 包含 targetURL → WebBrowserManager 打开 URL
    │   └── mode 字段决定展示方式（normal / immersive / modal）
    │
    ├── payload 包含 targetAppId → 查找本地缓存
    │   ├── 找到缓存 → 打开缓存的离线应用
    │   ├── AppId 是有效 URL → 直接打开
    │   └── 未找到 → 提示 "未找到应用缓存"
    │
    └── 无路由目标 → 仅标记已读，不做跳转
```

## 5. 发现/应用页

### 5.1 已缓存应用网格

此功能集成在首页的卡片视图中，不设独立 Tab。首页卡片即为已缓存应用的展示入口。

```
┌─────────────────────────────────────────┐
│  ─ 已缓存应用 ─────────────────────────  │  ← Section Header

```
